# APEC AI 特别行动队 — H5 小游戏实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement.

**Goal:** 构建一个单文件 H5 像素风互动小游戏，5屏流程，点击交互，Canvas海报导出

**Architecture:** 单文件 HTML，CSS/JS 内嵌。状态机驱动5屏切换。所有素材路径集中在 JS 常量区，HTML注释标记替换点。

**Tech Stack:** 纯 HTML + CSS + vanilla JS，零依赖

---

## 文件结构

```
APEC-ai-game/index.html   ← 唯一的代码文件
```

index.html 内部结构：
- `<style>` 块 — 全部CSS
- `<body>` 块 — 5个 screen div + 1个隐藏 canvas
- `<script>` 块 — 全部JS（配置常量 → 状态管理 → 屏渲染 → 计分 → 海报）

---

### Task 1: HTML 骨架 + CSS 基础

**目标:** 搭建文件框架，完成像素风基础样式

- [ ] 创建 `index.html`，包含 viewport meta、5个 screen div、canvas
- [ ] 写入基础 CSS：像素字体、屏幕切换 `.hidden`、移动端优先容器、按钮样式
- [ ] 写入素材路径配置常量（顶部注释区，标记素材替换位置）

### Task 2: Screen 1 — 接单动画

**目标:** 消息展示 + 接单按钮 → 进入 Screen 2

- [ ] HTML：消息气泡结构、APEC消息内容、"接单！"按钮
- [ ] CSS：消息气泡样式、打字动画（可选，先做静态展示保证稳定）
- [ ] JS：点击接单 → 隐藏 Screen1 → 显示 Screen2

### Task 3: Screen 2 — 组队选人（核心玩法）

**目标:** 4张人物卡，点击选中最多3人，确认组队

- [ ] HTML：4张人物卡（img + 姓名 + 属性标签）、确认按钮
- [ ] CSS：卡片网格布局、选中态（像素边框高亮）、未选中态（半透明）
- [ ] JS：点击选中/取消（限制3人）、选满3人启用确认按钮、确认→进入Screen3

### Task 4: Screen 3 — 危机事件（3轮）

**目标:** 随机顺序3个事件，每次3选1，属性加成计分

- [ ] HTML：事件描述区、3个选项按钮
- [ ] CSS：事件卡片样式、选项按钮样式
- [ ] JS：事件数组（带属性标记）、shuffle顺序、逐轮推进、计分累加

### Task 5: Screen 4 — 交付动画

**目标:** 自动播放序列（装车→地标→会场→机器人说话→评分）

- [ ] HTML：动画内容区、切换图片展示区
- [ ] CSS：淡入淡出过渡
- [ ] JS：setTimeout 驱动4个阶段自动切换、最后自动进入 Screen5

### Task 6: Screen 5 — 得分 + Canvas 海报

**目标:** 显示称号、生成海报、重玩/分享

- [ ] HTML：称号展示区、重玩按钮、分享海报按钮
- [ ] JS：根据总分计算称号、Canvas 绘制海报（背景+称号+图标）、导出图片触发下载

### Task 7: 桌面适配 + BGM 控制 + 收尾

**目标:** 桌面端居中、BGM开关、全局测试

- [ ] CSS：桌面端 media query、max-width 居中
- [ ] HTML：BGM 开关按钮
- [ ] JS：BGM 播放/暂停控制
- [ ] 全流程测试：Screen1→5 走通、计分准确、海报正常导出
